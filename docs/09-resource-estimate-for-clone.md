# 09 · 做 Raft 平替需要投入多少资源

> 问题：做一个 **Raft 平替**（coforge，开发初期代号 slock），要投入多少资源？
>
> **边界定义**："平替"= 复制 Raft 的核心体验（C1-C5 五能力够用即可），**不追求超越**（不含 [04] 的差异化层、不含 [08] 的绕路 feature）。即：一个能自托管、人机共享频道、named agent 有持久记忆、多 agent 基础协调、开箱即用的产品，体验上能替代 Raft 给小团队用。
>
> **结论先行**（复用开源基线）：
> - **人力**：5 人核心团队，6-9 个月到"可用平替"，10-12 个月到"成熟平替"
> - **资金**：~$500k-900k 跑到可用版；~$900k-1.5M 跑到成熟版（含人力 + 基础设施 + LLM）
> - **关键前提**：复用开源（LibreChat + Letta/LangGraph + OpenHands），C3 只做轻量版
>
> 全自研：人力 ×1.5-2，资金 ×1.6-2.2。下文是拆解。

---

## 一、先界定"平替"的 MVP 范围（C1-C5 够用版）

| 能力 | 平替 MVP 要做到 | 不做（省掉） |
|------|----------------|------------|
| C1 聊天工作区 | channel/DM/thread、人机共享、@mention 路由 | 多视图（board/graph/diff/timeline） |
| C2 持久记忆 | agent 跨会话存活、基础记忆积累 | 记忆可检视/diff/provenance/branch/ACL |
| C3 多 agent 协调 | 轻量锁 + 串行化 + 基础 inbox + held draft | 原生并发原语、事件流房间、能力寻址 |
| C4 自托管 daemon | Docker 自部署、agent 跑本地 | air-gap、企业 VPC 细节 |
| C5 成品体验 | 开箱即用、有 onboarding、基础可观测 | 企业 SSO/RBAC、计费、marketplace |

**这正好是 [07] 的 Phase 0 + Phase 1 的"轻量 C3"子集**。所以可以复用 [07] 的模块拆解，只是把 C3 从"原生并发"降级到"轻量够用"。

## 二、人力投入

### 团队配置（5 人核心，最经济可行的平替配置）

| 角色 | 人数 | 职责 | 月成本（含 overhead，美元） |
|------|------|------|---------------------------|
| 全栈/架构师（Tech Lead） | 1 | 架构、C3 协调层、系统集成 | $18-25k |
| 后端工程师 | 1 | daemon、记忆接入、inbox/held draft | $14-20k |
| 前端工程师 | 1 | 聊天 UI 改造（LibreChat fork）、onboarding | $14-20k |
| Agent/LLM 工程师 | 1 | prompt、agent runtime、记忆策略 | $14-20k |
| DevOps/兼职 | 0.5 | 部署、CI、可观测 | $7-10k |
| **合计** | **4.5 人** | | **~$67-95k/月** |

> 加一个产品/设计 0.5 人更稳，但平替阶段可由 Tech Lead 兼。硅谷之外成本可降 30-50%。

### 阶段人力（人月）

复用 [07] 模块，但 C3 降级为轻量版：

| 阶段 | 模块 | 人月 |
|------|------|------|
| Phase 0 PoC | M1 共享频道改造、M2 持久身份接入、M3 基础 inbox、M4 命名路由、M13 daemon | 7-10 |
| Phase 1 轻量 C3 | M5 held draft、M6 轻量锁+串行化（降级版）、M7 跳过/最小化（用快照轮询替事件流） | 5-8 |
| 产品化 | onboarding、错误处理、基础可观测、文档 | 4-6 |
| **平替可用版合计** | | **16-24 人月** |
| 打磨加固 | 稳定性、性能、更多 vertical 适配 | 6-10 |
| **成熟平替合计** | | **22-34 人月** |

→ 5 人团队，自然月：**可用版 4-5 个月**（但关键路径串行，实际 **6-9 个月**）；**成熟版 10-12 个月**。

> 与 [07] 对比：[07] 追平 Raft 底座是 6-9 个月、超越是 12-18 个月。这里"平替可用"≈ [07] 的 Phase 0+1 轻量版，6-9 个月一致；"成熟平替"加产品化打磨，10-12 个月。

## 三、资金投入（跑到可用版 / 成熟版）

### 可用平替版（6-9 个月）

| 项 | 金额（美元） |
|----|------------|
| 人力（5 人 × 7.5 月均） | $500-710k |
| LLM API（开发+测试+dogfooding） | $15-30k |
| 基础设施（云、CI、存储） | $5-10k |
| 工具/服务（监控、错误追踪等） | $3-5k |
| 法务/公司运营（如开源可省） | $2-5k |
| 缓冲 15% | $80-115k |
| **合计** | **~$600-875k** |

### 成熟平替版（10-12 个月）

| 项 | 金额（美元） |
|----|------------|
| 人力（5 人 × 11 月均） | $740-1.05M |
| LLM API（含早期用户用量） | $30-60k |
| 基础设施 | $10-15k |
| 工具/服务 | $5-8k |
| 法务/运营 | $3-8k |
| 缓冲 15% | $120-175k |
| **合计** | **~$900k-1.3M** |

> 硅谷之外（如中国/东欧/拉美）人力成本降 30-50%：可用版 ~$350-550k，成熟版 ~$600-900k。
> 全自研不复用开源：人力 ×1.5-2，总额 ×1.6-2.2（可用版 ~$1-1.7M，成熟版 ~$1.5-2.6M）。

## 四、时间投入

| 里程碑 | 自然月（5 人） | 产出 |
|--------|--------------|------|
| M2 内部 demo | 2-3 | 共享频道 + named agent + 持久记忆能跑 |
| M4 可用平替（alpha） | 4-5 | 轻量协调、自托管、小团队可试 |
| **M6-9 平替可用版** | 6-9 | 产品化完成，能替代 Raft 给小团队用 |
| **M10-12 成熟平替** | 10-12 | 稳定、性能 OK、多 vertical 适配 |

## 五、不同资源档位的产出对比

| 档位 | 团队 | 时间到可用 | 资金 | 能平替到什么程度 |
|------|------|-----------|------|----------------|
| **极简** | 2-3 人 | 10-14 月 | $250-450k | 能出 PoC，平替度 ~60%，难成熟 |
| **经济**（推荐） | 5 人 | 6-9 月 | $600-875k | 平替可用版 ~80-85% |
| **标准** | 8 人 | 5-7 月 | $900k-1.3M | 平替 ~85-90%，成熟度更高 |
| **重投入** | 12 人 | 4-6 月 | $1.3-2M | 平替 ~90%+，接近追平 Raft |
| 全自研重投入 | 12 人 | 8-12 月 | $2-3M | 同上但不复用开源 |

**关键拐点**：5 人是性价比拐点。低于 5 人，关键路径串行导致时间成本暴涨；高于 8 人，M6/M7 类正确性任务加人边际递减（Brooks 定律），且 C3 已降级不需要那么多人。

## 六、复用开源省了多少资源

| 若全自研 | vs 复用开源 | 省的比例 |
|---------|-----------|---------|
| 自造 chat UI（~18-25pm） | fork LibreChat（~2-3pm） | C1 省 ~80% |
| 自造记忆引擎（~8-12pm） | 接 Letta/LangGraph（~1-2pm） | C2 省 ~80% |
| 自造 daemon runner（~6-8pm） | 借 OpenHands agent-server（~1-2pm） | C4 省 ~70% |
| C3 协调（无开源，省不了） | 自研轻量版 | 0% |
| 产品化（无开源） | 自研 | 0% |
| **总额** | | **省 ~35-45%** |

→ 复用开源把"5 人 6-9 月 $600-875k"压到这个数；全自研要 ~$1-1.7M 才到可用版。**复用开源是平替的经济性前提**。

## 七、最大风险与隐藏成本

1. **缝合成品的接缝工程**（+3-5 人月，+~$30-60k）：LibreChat ↔ Letta ↔ C3 三套抽象缝合的一致性问题，容易被低估。这是 [06] "组合到 70-80%、剩 20-30% ≈ 自研"的具体成本。
2. **C3 降级版的体验落差**：轻量锁+串行化在 agent 多了之后会撞车（计数游戏到 5 就乱）。平替能接受"小团队够用"，但无法在协调体验上替代 Raft——这是"平替"与"替代"的差距，要诚实告知用户。
3. **LLM 成本不可控**：一旦真有用户，token 成本可能爆（Raft 用户峰值 1.2B tokens/day）。平替阶段要内置 token 预算控制 + BYOK（用户自带 key），否则 $30-60k 的 LLM 预算挡不住。
4. **维护成本**：fork 的开源项目（LibreChat 等）持续 upstream，不跟会 drifted，跟要持续投入——长期 +1-2 人月/月 的维护。
5. **产品化打磨**：从"能跑"到"别人能用"，onboarding/错误提示/文档通常 +20-30% 工期，已在上面计入但常被低估。

## 八、给决策的建议

1. **若目标是"自用 + 开源一个平替"**：2-3 人、$250-450k、10-14 个月能出 PoC 级平替（~60%）。适合作为开源项目而非商业产品。
2. **若目标是"给小团队的可用平替产品"**：5 人、$600-875k、6-9 个月（推荐档位）。这是性价比最优解。
3. **若目标是"商业级平替、抢 Raft 用户"**：8 人、$900k-1.3M、5-7 个月到可用，再 3-6 个月成熟。但注意——纯平替没有差异化，会和 Raft 正面撞，且 Raft 有叙事+先发优势。**纯平替商业上风险高，建议至少带一个 [08] 的绕路 feature 做差异化**（如 + 可迁移记忆 或 + agent 观测），多投 ~$100-200k 换一个能站住的定位。
4. **若资源紧张**：走 [08] 组合 3（协议 + 单一 vertical），~$350-550k、6-8 个月，避开 C3，且天然有差异化——比纯平替更可能活下来。

---

## 一句话

> 做 Raft 平替（复用开源 + C3 轻量版），**5 人团队 6-9 个月、约 $600-875k 到可用版；10-12 个月、约 $900k-1.3M 到成熟版**。全自研 ×1.6-2.2。复用开源省 ~35-45%（主要在 C1 外壳 + C2 记忆底座 + C4 daemon），C3 和产品化省不了。性价比拐点是 5 人。但纯平替商业上会和 Raft 正面撞且无差异化，建议至少带一个绕路 feature（多投 ~$100-200k）换定位——否则平替易做，活下来难。
