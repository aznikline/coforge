# 10 · Claude Code 直接做要多少资源

> 问题：**你（Claude Code）直接做** Raft 平替，要多少资源？
>
> 这和 [09]（5 人团队）是完全不同的口径。没有团队工资，核心成本是 **token + 时间 + 我的能力边界**。下文是诚实的拆解，包括我能做什么、做不了什么。

---

## 一、先说硬约束（否则数字会骗你）

"我直接做"不等于"我一个人等于一个团队"。有几个结构性约束必须摆明：

1. **没有持续运行的环境**。我能写代码、跑命令、测试，但**会话结束我的执行就停**。一个需要"持续运行 daemon + 长期 memory store + 实时事件流"的产品，它的运行态我不在线守护——我能造出来，不能"住"在里面运维。这恰好是 Raft 类产品的核心形态（持久 agent 跨天存活）。
2. **LLM 成本仍要你出**。我做出来的产品里，agent 调用 LLM 的 token 费用是**你的 API key**，不是我。我能省的是"开发过程的 token"，不是"产品运行的 token"。
3. **我能并行写代码，但不能并行"人"**。我可以一次起多个 subagent 并行写不同模块，但**C3 这种正确性串行任务**，我加再多 subagent 也不会更快——分布式 bug 我也得一个一个调（见 [07] 的 Brooks 定律分析，对我同样成立）。
4. **设计判断我能做，但产品方向你要拍板**。[08] 的 7 条路线选哪条、平替还是超越、目标用户是谁——这些是产品决策，不是代码问题。我可以给推荐，但错了你买单。
5. **我能复用开源，省的和你自己复用一样**。fork LibreChat、接 Letta——这些开源红利对我同样有效，[09] 省的 ~35-45% 我也省。

**一句话**：我能把"5 人 6-9 个月 $600-875k"的开发人力部分**几乎归零**（变成我的 token 费 + 你的时间），但**运行成本、基础设施、LLM 用量、产品决策这些我替代不了**。

## 三、"我直接做"的资源拆解

### 开发人力成本：≈ $0（变成 token 费）

这是最大的省。[09] 的 5 人 × 6-9 月 = $500-710k 人力，**我直接做这部分归零**，只产生 Claude Code 的 token 费用。

### 我这边的 token 费 + 时间（开发过程）

| 里程碑 | 我的工时（连续会话） | token 费（估） | 你的墙钟时间 |
|--------|---------------------|--------------|-------------|
| PoC（共享频道 + named agent + 持久记忆能跑） | ~30-50 小时 | $200-500 | 2-5 天（若你连续驱动） |
| 可用平替版（轻量 C3 + 自托管 + 产品化） | ~120-200 小时 | $1-3k | 2-4 周（连续）/ 6-10 周（间歇） |
| 成熟平替版 | ~250-400 小时 | $3-8k | 5-10 周（连续）/ 3-6 月（间歇） |

> token 费是粗估——取决于用 Sonnet/Opus、上下文长度、subagent 数量。用 Opus 做架构 + Sonnet 做实现是性价比组合。
> "连续会话"指你持续驱动我往前推；"间歇"指每天推几小时——**墙钟时间主要取决于你的驱动节奏，不是我的速度**。

### 你必须出的资源（我替代不了的部分）

| **你的时间**（驱动我 + 拍板 + 测试反馈） | 50-200 小时 | 这是最大隐性投入——我不是自动机，要你持续驱动和验收 |
| 企业能力（若要） | 你或第三方 | SSO/RBAC/审计/合规认证我写代码，但 SOC2 认证要机构做 |

### 总账（我直接做到可用平替版）

| 项 | 成本 |
|----|------|
| 开发人力 | **~$0**（省 $500-710k） |
| 我的 token 费 | $1-3k |

**对比 [09] 的 $600-875k**：我直接做把一次性开发投入从六十万级压到几千美元级。**这是数量级的差距。**

## 四、为什么这么便宜，以及代价是什么

省钱的本质：**Claude Code 把"写代码"这个最贵的劳动边际成本归零了**。5 人团队 6-9 月的工资，是我几十到几百小时的 token。

但代价很真实：

1. **质量风险**。我能写出来，但"能用"到"稳定"之间的打磨、边界 case、长期运行回归——我需要你反复测试反馈，或靠 CI 长期守。无人 review 的 AI 生成代码，长期维护性存疑。
2. **C3 仍是天花板**。即使我写，M6/M7 的分布式正确性 bug 我也得慢慢调，且会话结束我得重新加载上下文。这部分不会因为我"免费"就变快。
3. **产品力 ≠ 代码**。Raft 的成功是叙事+设计+工程审美的对齐（[02]）。我能写工程，但**叙事、品牌、设计身份、用户研究**这些要你做或另请人——这是"平替易做、活下来难"的根因（[09] 已警告）。
4. **我的连续性有限**。长任务跨会话要靠 memory + 文档重建上下文，每次有损耗。复杂调试可能要我反复"重新理解代码"。
5. **你的时间是真实瓶颈**。我不是自动机——给个需求就自己跑半年。每一步要你驱动、验收、拍板。50-200 小时你的时间，是硬约束。

| 可用平替 | ~$1-3k token + 120-200h 我 | ~50-150h 你 + 月度运行费 | 替代 Raft 给小团队，~80% |

| 超越 Raft（完整 [04]） | ~$5-15k token + 400-600h 我 | ~200-400h 你 + 持续投入 | 功能超越，但需长期迭代 |

## 六、诚实的建议

1. **若你想验证"能不能做"**：让我先做 PoC（~$200-500 token + 你 10-20 小时）。2-5 天能看到能跑的共享频道 + named agent。这是最低成本的下一步。
2. **若你想真做一个平替产品**：我直接做开发能省 ~$500-710k 人力，但**你要补上产品/叙事/用户研究/运维**——否则出来的只是"代码能跑"，不是"产品能活"。建议预算里留 $50-150k 给设计、品牌、早期用户获取，这比代码更决定成败。
3. **若你想超越 Raft**：代码我能写，但 [08] 的路线选择、[02] 的叙事占位、企业销售——这些是人的决策和关系，我替代不了。代码只占超越成本的 ~30%，其余 70% 是产品+市场+信任。
4. **最大风险提醒**：免费代码会让人低估难度。Raft 的护城河是 C3 工程内功（[02] 因素 4）+ 叙事占位（因素 1）+ 设计身份（因素 8）——我能写 C3 的轻量版，但**叙事和设计身份我给不了**。纯靠我写代码做出的平替，会缺 Raft 最值钱的那两层。
